# [대시보드 Sprint v0.1] "⚡ 생각 발전소" 탭 신설

## 🚦 씽크 협의 결과 (2026-04-11 확정)

**씽크 클과장 답변**: [.claude/inbox/2026-04-11_씽크_생각발전소_v0.1_답변.md](.claude/inbox/2026-04-11_씽크_생각발전소_v0.1_답변.md)

### 반드시 알아야 할 3가지

1. **status 필드는 신규 생성이 아님 — 전이만 함**
   - bot.py(양념이)가 파일 저장 시 이미 `status: 미검토`를 자동으로 박음
   - 대시보드는 사용자 액션 시 `미검토` → `revive|realized|archived` **1회성 전이**만 수행
   - 새 파일에 status 필드를 생성하는 코드 작성하지 말 것
2. **추첨 대상 필터**
   - `status == "미검토"` 또는 `status` 필드 자체가 없는 파일만 추첨 풀에 포함
   - `revive` / `realized` / `archived` 는 추첨 풀에서 **제외**
3. **Think 시스템 충돌 없음 확정**
   - bot.py는 status 필드를 읽지 않음 (코드 레벨 확인 완료)
   - 주간 회고, 통계, 검색도 status 미사용
   - 필드명 `lifecycle`로 바꿀 필요 없음. `status` 그대로 사용.

## 🎯 목표 한 줄

심심할 때 들어와서 주사위(🎲) 굴리며 과거 생각을 소비하는 **놀이터 탭** 신설. 단순 엔터테인먼트가 아니라 **능동 큐레이션 루프** 만들기.

## 🧠 컨셉 (왜 만드는가)

- 기존 대시보드는 **업무 도구**였음. inbox → 분류 → 저장. 쌓이기만 하고 재발굴이 없음.
- 후추님 6년치 Think Tank 자산을 **엔터테인먼트 형태로 소비** (틱톡·릴스 UX를 자기 지식에 적용).
- 사용 맥락: 엘리베이터 기다릴 때, 화장실, 지하철. "뭐 재밌는 거 없나" 모드.
- **리스크 방어**: 단순 스쳐보기로 끝나면 킬링타임 앱이 됨 → **액션 버튼 4개**로 큐레이션 유도.

## ✅ Pass 기준 (모두 충족해야 완료)

1. 하단 탭바 **맨 오른쪽(stats 뒤, 7번째)**에 `<i data-lucide="zap">` 아이콘 탭 추가. 라벨: "생각 발전소"
2. 탭 진입 시 카드 1개 + 🎲 버튼이 화면 중앙
3. 카드 내용: 경과일("3개월 전"), 제목, 태그, 확신도 이모지, 본문 첫 2~3줄
4. 추첨 범위: **30일 / 90일 / 180일 (±7일)** 구간에서 랜덤. 구간별 uniform 선택.
   - **추첨 대상 필터**: `status == "미검토"` 또는 `status` 필드 없는 파일만 포함
   - `revive` / `realized` / `archived` 는 추첨 풀에서 **반드시 제외**
5. **세션 유지**: `sessionStorage`에 현재 카드 경로 저장. 뒤로가기/새로고침 시 동일 카드 유지.
6. 🎲 버튼 → 다음 항목으로 교체. **같은 세션에서 본 항목은 제외** (sessionStorage에 본 경로 배열).
7. 모바일 레이아웃 안 깨짐. 기존 6개 탭 간격 조정 포함 (탭 7개 수용).
8. 다크모드에서도 카드 가독성 OK.
9. **XSS 방지**: 제목·본문·태그 모두 HTML escape 후 출력. `innerHTML` 사용 시 escape 필수.
10. **액션 버튼 4개 모두 구현** (모두 `status` 값 전이 방식):
    - 🔥 **다시 살리기** → frontmatter `status: 미검토` → `revive` 로 전이 (v0.1은 전이만, 실제 파일 이동은 v0.2)
    - 📌 **북마크** → `localStorage`에 path 저장 (status 필드 안 건드림, 모음 뷰는 v0.2)
    - ✅ **이미 했음** → frontmatter `status: 미검토` → `realized` 로 전이, 추첨 풀 제외
    - 🗑️ **그만 봄** → frontmatter `status: 미검토` → `archived` 로 전이, 추첨 풀 제외
    - **중요**: 새 status 필드를 만드는 게 아님. 기존 값(`미검토`)을 바꾸는 것.
11. 해당 범위에 추첨 가능한 파일 0개 → "오늘은 발전할 연료가 없네요 ⚡" 문구 (카드 자리 유지)
12. Playwright 또는 실기기로 모바일 렌더링 확인
